# 02-需求规格说明书(SRS).md

## 一、引言

### 1.1 编写目的

本文档旨在明确**基于微信小程序的校园服务智能问答系统**的功能需求与非功能需求，为系统设计、开发、测试提供依据。本文档主要面向以下读者：

- **项目组成员**：明确开发目标和范围
- **测试人员**：制定测试用例和验收标准
- **导师**：评审需求完整性与合理性
- **毕业设计答辩组**：展示需求分析过程

### 1.2 项目范围

本项目开发校园服务智能问答系统，涵盖以下核心范围：

| 范围项       | 包含内容                           | 不包含内容                |
| ------------ | ---------------------------------- | ------------------------- |
| **用户角色** | 学生、部门信息员、系统管理员       | 游客匿名使用核心功能      |
| **前端入口** | 微信小程序（学生端）               | 独立APP、H5页面           |
| **管理后台** | Web管理后台（Vue.js + Element UI） | 移动端管理后台            |
| **智能问答** | Jieba分词 + 关键词匹配 + 分类检索  | 深度学习模型、NLP语义理解 |
| **信息聚合** | 通知公告、办事指引、知识库         | 第三方教务系统实时对接    |
| **消息推送** | 微信订阅消息（模拟）               | 短信推送、APP推送         |
| **支付功能** | 无                                 | 真实支付功能              |

### 1.3 术语表

| 术语         | 说明                                             |
| ------------ | ------------------------------------------------ |
| **Jieba**    | 中文分词工具，用于对用户提问进行分词处理         |
| **知识库**   | 存储问答对的数据集合，包含问题、答案、分类等信息 |
| **信息员**   | 各部门负责维护本部门问答内容和通知公告的管理人员 |
| **订阅消息** | 微信小程序提供的消息推送能力                     |
| **RBAC**     | 基于角色的访问控制（Role-Based Access Control）  |
| **JWT**      | JSON Web Token，用于用户身份认证                 |
| **意图识别** | 通过规则匹配识别用户查询意图的技术               |
| **置信度**   | 系统对答案匹配程度的量化评分（0-1之间）          |

### 3.2 核心业务用例详述

#### UC-01：智能提问

| 项目         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| **用例名称** | 智能提问                                                     |
| **参与者**   | 学生                                                         |
| **前置条件** | 学生已登录微信小程序                                         |
| **后置条件** | 系统返回答案，记录查询历史                                   |
| **基本流程** | 1. 学生进入小程序首页<br>2. 在搜索框输入问题（如"图书馆几点开门"）<br>3. 系统使用Jieba进行分词处理<br>4. 系统匹配知识库中的问答对<br>5. 系统计算匹配置信度<br>6. 返回答案及相关推荐问题<br>7. 学生查看答案并可评价 |
| **扩展流程** | 3a. 分词失败：使用原始文本匹配<br>4a. 无匹配结果：返回相似问题推荐或联系信息<br>6a. 置信度低于阈值：提示"未找到精确答案"并推荐相关问题 |
| **业务规则** | 1. 匹配阈值默认为0.6<br>2. 单次查询返回最多5个相关问题<br>3. 查询历史保存最近20条 |

#### UC-02：问答知识库管理

| 项目         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| **用例名称** | 问答知识库管理                                               |
| **参与者**   | 部门信息员                                                   |
| **前置条件** | 信息员已登录Web管理后台                                      |
| **后置条件** | 问答数据更新，同步到搜索索引                                 |
| **基本流程** | 1. 信息员进入"知识库管理"页面<br>2. 点击"新增问答"<br>3. 填写问题、答案、选择分类<br>4. 设置关键词（可选）<br>5. 选择是否标记为热点<br>6. 提交保存<br>7. 系统自动更新分词索引 |
| **扩展流程** | 3a. 问题已存在：提示重复，建议编辑<br>4a. 未设置关键词：系统自动从问题中提取<br>6a. 提交失败：提示错误信息，保留表单数据 |
| **业务规则** | 1. 问题长度限制5-200字<br>2. 答案长度限制10-2000字<br>3. 每个信息员只能管理本部门问答<br>4. 新增问答需审核后生效（可配置） |

#### UC-03：发布通知公告

| 项目         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| **用例名称** | 发布通知公告                                                 |
| **参与者**   | 部门信息员                                                   |
| **前置条件** | 信息员已登录且具有发布权限                                   |
| **后置条件** | 通知发布成功，订阅用户收到推送                               |
| **基本流程** | 1. 信息员进入"通知管理"页面<br>2. 点击"发布通知"<br>3. 填写标题、内容、摘要<br>4. 选择发布时间（立即/定时）<br>5. 选择目标范围（全校/特定院系）<br>6. 上传附件（可选）<br>7. 预览并确认发布<br>8. 系统向订阅用户发送订阅消息 |
| **扩展流程** | 3a. 内容含敏感词：提示修改<br>5a. 选择特定院系：弹出院系选择器<br>7a. 选择定时发布：设置发布时间，进入待发布列表 |
| **业务规则** | 1. 标题长度限制10-100字<br>2. 正文支持富文本编辑<br>3. 附件大小限制10MB<br>4. 每人每天最多发布10条通知 |

2. 支持按用户、时间、操作类型筛选
3. 日志保留90天
4. 支持导出审计报告
5. 异常操作自动告警（如频繁登录失败）

作为[系统管理员]，我想要[监控系统运行状态]，以便[及时发现性能问题]
验收标准：
1. 展示系统访问量、QPS、响应时间
2. 展示数据库连接池、缓存命中率
3. 展示错误率、慢查询统计
4. 支持设置告警阈值
5. 数据可视化展示（图表）

## 六、非功能性需求

### 6.1 性能需求

| 需求项           | 指标要求            | 验证方法               |
| ---------------- | ------------------- | ---------------------- |
| **搜索响应时间** | 平均<1.5秒，95%<2秒 | JMeter压力测试         |
| **分词处理时间** | <500ms              | 单元测试               |
| **并发支持**     | 支持100用户同时查询 | JMeter模拟100并发      |
| **页面加载时间** | 小程序首页<2秒      | 微信开发者工具性能面板 |
| **数据库查询**   | 简单查询<100ms      | MySQL慢查询日志        |
| **通知推送延迟** | 从发布到接收<5分钟  | 模拟测试               |

### 6.2 安全性需求

| 需求项         | 要求                 | 实现方式                 |
| -------------- | -------------------- | ------------------------ |
| **用户认证**   | 微信授权 + JWT Token | Spring Security + JWT    |
| **接口安全**   | 防止未授权访问       | 接口鉴权、Token过期机制  |
| **数据加密**   | 敏感字段加密存储     | AES-256加密              |
| **防SQL注入**  | 参数化查询           | MyBatis-Plus预编译       |
| **防XSS攻击**  | 输入过滤             | 前端过滤 + 后端转义      |
| **敏感词过滤** | 实时检测违规内容     | DFA算法敏感词过滤        |
| **操作日志**   | 关键操作可追溯       | AOP记录日志              |
| **数据备份**   | 每日自动备份         | MySQL定时备份 + 异地存储 |

### 6.3 可用性需求

| 需求项         | 要求                           | 验证方法     |
| -------------- | ------------------------------ | ------------ |
| **系统可用性** | 核心功能7×24小时可用           | 监控报警     |
| **容错性**     | 单点故障不影响核心功能         | 服务降级测试 |
| **数据一致性** | 知识库更新后搜索立即生效       | 功能测试     |
| **兼容性**     | 支持微信iOS/Android最新3个版本 | 真机测试     |

### 6.4 易用性需求

| 需求项         | 要求                     | 实现方式          |
| -------------- | ------------------------ | ----------------- |
| **操作引导**   | 新用户首次使用有引导提示 | 新手引导组件      |
| **搜索体验**   | 支持模糊匹配、拼音搜索   | 拼音分词插件      |
| **响应式设计** | 适配主流手机屏幕尺寸     | rpx自适应布局     |
| **错误提示**   | 用户友好的错误信息       | 统一错误码 + 文案 |
| **帮助文档**   | 提供使用说明和常见问题   | 内置帮助中心页面  |

### 6.5 可维护性需求

| 需求项         | 要求                 | 实现方式             |
| -------------- | -------------------- | -------------------- |
| **代码规范**   | 统一编码风格         | ESLint + Alibaba规范 |
| **文档完整性** | 关键模块有设计文档   | Markdown文档维护     |
| **日志分级**   | DEBUG/INFO/ERROR分级 | Logback配置          |
| **监控告警**   | 关键指标异常告警     | 自定义监控脚本       |
| **模块化**     | 业务模块低耦合       | 分层架构设计         |
